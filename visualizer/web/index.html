Honesty notes: SM tiles show block residency (from the block→SM trace), not per-SM counters. L2 / DRAM / PCIe values are throughput rates — cache fill level is not measurable on real hardware. The only true capacity gauges are VRAM and warp slots. Rate lanes are device-wide averages; under colocation they blend both tenants. NVLink absent on this GPU. Colors: on SM tiles a color = a kernel; on lanes/gauges/numbers a color = a metric (the two domains never share a panel; 8-slot palette is reused across domains). The GPC grouping is schematic: 170 SMs are real (CUDA device query), and GB202 really has 12 GPCs — but the true per-GPC SM distribution is per-die yield harvesting (11 TPCs fused off, pattern unpublished and not queryable), so tiles are dealt into boxes evenly for display only.
